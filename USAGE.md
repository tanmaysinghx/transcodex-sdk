# TranscodeX SDK Usage Guide

This guide describes how to integrate the TranscodeX SDK into your Java applications and use its core APIs.

---

## 1. Adding Dependencies

To use TranscodeX, add the API contract and FFmpeg implementation modules to your project's `pom.xml`:

```xml
<dependencies>
    <!-- Public API Contracts and Core Domain Models -->
    <dependency>
        <groupId>io.transcodex</groupId>
        <artifactId>transcodex-api</artifactId>
        <version>0.1.0-SNAPSHOT</version>
    </dependency>

    <!-- FFmpeg/FFprobe Command-line Execution Implementations -->
    <dependency>
        <groupId>io.transcodex</groupId>
        <artifactId>transcodex-ffmpeg</artifactId>
        <version>0.1.0-SNAPSHOT</version>
    </dependency>
</dependencies>
```

---

## 2. System Prerequisites

The execution layer communicates directly with native binaries on your system. Make sure `ffmpeg` and `ffprobe` are:
1. Installed on the host machine.
2. Added to the system environment `PATH` variable.

*Tip: You can verify this by running `ffmpeg -version` and `ffprobe -version` inside your terminal.*

## 3. Usage Examples

### A. Extracting Video Metadata

Here is how to extract play duration, streams, codecs, and container formats from a media file:

```java
import io.transcodex.api.metadata.MetadataExtractor;
import io.transcodex.core.metadata.VideoMetadata;
import io.transcodex.ffmpeg.executor.FfmpegExecutor;
import io.transcodex.ffmpeg.executor.ProcessBuilderExecutor;
import io.transcodex.ffmpeg.metadata.FfprobeMetadataExtractor;

import java.nio.file.Path;

public class MetadataExample {

    public static void main(String[] args) {
        // 1. Initialize process executor (uses Virtual Threads for low overhead)
        FfmpegExecutor executor = new ProcessBuilderExecutor();

        // 2. Initialize extractor (defaults to querying "ffprobe" on system path)
        MetadataExtractor extractor = new FfprobeMetadataExtractor(executor);

        // Alternatively, specify custom path to your ffprobe binary:
        // MetadataExtractor extractor = new FfprobeMetadataExtractor(executor, "C:/tools/ffmpeg/bin/ffprobe.exe");

        try {
            Path videoFile = Path.of("input_video.mp4");
            
            // 3. Run extraction
            VideoMetadata metadata = extractor.extract(videoFile);

            // 4. Access parsed container metadata
            System.out.println("=== Format Details ===");
            System.out.println("Container Format: " + metadata.format());
            System.out.println("Duration: " + metadata.duration().toSeconds() + " seconds");
            System.out.println("File Size: " + metadata.sizeBytes() + " bytes");
            if (metadata.bitrateBps() != null) {
                System.out.println("Bitrate: " + (metadata.bitrateBps() / 1000) + " kbps");
            }

            // 5. Access video track metadata
            var video = metadata.videoStream();
            System.out.println("\n=== Video Stream ===");
            System.out.println("Codec: " + video.codec());
            System.out.println("Dimensions: " + video.width() + "x" + video.height());
            System.out.println("Frame Rate: " + video.frameRate() + " fps");
            System.out.println("Aspect Ratio: " + video.aspectRatio());

            // 6. Access optional audio track metadata
            metadata.audioStream().ifPresent(audio -> {
                System.out.println("\n=== Audio Stream ===");
                System.out.println("Codec: " + audio.codec());
                System.out.println("Channels: " + audio.channels());
                System.out.println("Sample Rate: " + audio.sampleRate() + " Hz");
            });

        } catch (Exception e) {
            System.err.println("Metadata extraction failed: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
```

---

## 4. Advanced: Configuration and Scalability

- **Virtual Threads Integration:** Since the underlying process stream readers in `ProcessBuilderExecutor` block on stream reads, running this block inside Java virtual threads (`Thread.startVirtualThread(...)` or standard Executors) yields the CPU carrier thread. This ensures massive processing concurrency without platform thread starvation.
- **Error Handling:** All exceptions generated by extraction are wrapped inside the unchecked `MetadataExtractionException` which provides clear details about exit codes and process `stderr` contents.
