# Security Policy

We take the security of TranscodeX SDK very seriously. If you believe you have found a security vulnerability, please report it to us using the instructions below.

---

## 1. Supported Versions

Security updates are actively applied to the following versions:

| Version | Supported | Notes |
| :--- | :---: | :--- |
| **0.1.x** | Yes | Active release cycle (Alpha/Beta stage) |
| **< 0.1.x** | No | Legacy preview builds |

---

## 2. Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.** Instead, follow these steps:

1. Send an email to **security@transcodex.io** with details of the vulnerability.
2. Include a description of the issue, a proof-of-concept (PoC), and steps to reproduce.
3. We will acknowledge your email within **48 hours** and provide a tracking ID.
4. We request that you follow **coordinated vulnerability disclosure (CVD)** guidelines, giving us reasonable time to fix the issue before public announcement.
